## Python 2.7.5 환경 호환성 규칙

본 프로젝트는 실운영 서버 환경인 Python 2.7.5 제약 조건하에서 파이썬 신규 및 수정 파이썬 소스 생성 및 구동하기 위한 구조 설계 및 코드 전환 규칙을 따릅니다.

### 1. 문자열 포맷팅 전환 (f-string 제거)

**Python 3 (불가):**
```python
f"{PROGRAM_NAME}_{last_dir}_{suffix}.csv"
```

**Python 2.7 (필수):**
```python
"{}_{}_{}.csv".format(PROGRAM_NAME, last_dir, suffix)
# 또는
"%s_%s_%s.csv" % (PROGRAM_NAME, last_dir, suffix)
```

### 2. 타입 힌팅(Type Hinting) 전면 제거

**Python 3 (불가):**
```python
def mysql_connect(conf: dict) -> tuple:
```

**Python 2.7 (필수):**
```python
def mysql_connect(conf):
    # conf: dict, return: tuple (주석으로 대체)
```

### 3. 파일 입출력 및 인코딩 처리

Python 2.7은 `open()` 함수의 `encoding` 파라미터를 미지원합니다.

**필수 패턴:**
```python
import codecs

# 파일 읽기
with codecs.open(file_path, 'r', encoding='utf-8') as f:
    data = f.read()

# 파일 쓰기 (BOM 처리)
with codecs.open(out_file, 'w', encoding='utf-8') as f:
    f.write(codecs.BOM_UTF8)
    f.write(content)
```

### 4. OS 모듈 예외 처리

**Python 3 (불가):**
```python
os.makedirs(OUT_DIR, exist_ok=True)
```

**Python 2.7 (필수):**
```python
if not os.path.exists(OUT_DIR):
    os.makedirs(OUT_DIR)
```

### 5. 예외 처리 문법

**Python 3 (불가):**
```python
except Exception as e:
```

**Python 2.7 (호환):**
```python
except Exception, e:  # 또는 'as' 키워드도 지원
```

## 운영 안정성 및 디버깅 규칙

### Silent Fail 금지

모든 예외 처리 마지막에 `raise` 또는 `sys.exit(1)` 명시:

```python
try:
    cursor.executemany(query, batch)
except Exception, e:
    sys.stderr.write("[ERROR] DB 적재 실패: %s\n" % str(e))
    raise e
```

### 디버깅 출력

- 모든 주요 단계마다 시각과 작업 내용 기록
- `print` (stdout)과 `sys.stderr.write` (stderr) 분리
- 완료 시 처리 파일 수, 매칭 행 수, DB 적재 건수 요약 리포트

## 코딩 시 필수 확인 사항

✓ f-string 사용 제거 → `.format()` 또는 `%` 포매팅으로 변환  
✓ 타입 힌팅 제거 → 주석으로 문서화  
✓ `codecs.open()` 사용 → 인코딩 명시  
✓ `os.makedirs()` 시 존재 확인 → 방어 코드 추가  
✓ 예외 처리 시 `raise` 또는 `sys.exit()` 명시  
✓ 메모리 효율 → `fetchmany()`, `executemany()`, 제너레이터 활용
