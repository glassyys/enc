방법 2 사용하지 않고 vs코드를 재설치해도 동일한데 vs코드 삭제를 완전히하고 재설치하면 code명령어를 사용가능할지?

VS Code를 재설치했는데도 동일하다면, 기존에 남아있던 잘못된 레지스트리나 환경 변수 찌꺼기 때문에 새 설치 프로그램이 PATH(환경 변수)를 제대로 업데이트하지 못했을 가능성이 큽니다.

결론부터 말씀드리면, VS Code를 완전히 삭제(찌꺼기 파일까지 제거)한 후 재설치하면 code 명령어가 다시 정상 작동할 확률이 매우 높습니다.

완벽하게 초기화한 후 깔끔하게 재설치하는 단계를 안내해 드릴게요. (※ 주의: 이 작업을 하면 기존 VS Code 설정과 확장 프로그램이 초기화되므로, 중요한 설정은 백업해 두시는 것이 좋습니다.)

1단계: VS Code 완전 삭제하기
일반적인 제어판 삭제로는 설정 파일이 남아있기 때문에, 아래 순서대로 직접 폴더를 지워주셔야 합니다.

제어판 -> 프로그램 추가/제어에서 Visual Studio Code를 찾아 삭제(제거)합니다.

키보드의 Win + R을 눌러 실행 창을 켜고, %appdata%를 입력 후 엔터를 누릅니다.

열린 폴더에서 Code라는 이름의 폴더를 찾아 삭제합니다. (기존 설정·단축키 등 삭제)

다시 Win + R을 누르고, %userprofile%을 입력 후 엔터를 누릅니다.

열린 폴더에서 .vscode라는 이름의 폴더를 찾아 삭제합니다. (설치했던 확장 프로그램 삭제)

2단계: 환경 변수(Path)에 남아있는 찌꺼기 제거
재설치 전에 시스템에 꼬여있는 VS Code 관련 경로를 한 번 청소해 줍니다.

Win + R -> sysdm.cpl 입력 후 엔터 -> [고급] 탭 -> [환경 변수]로 이동합니다.

상단의 [사용자 변수]와 하단의 [시스템 변수] 양쪽 모두에서 Path 항목을 각각 더블 클릭해 엽니다.

목록 중에 Microsoft VS Code 또는 Code 관련 경로가 남아있다면 선택하고 [삭제]를 누른 뒤 확인을 눌러 저장합니다.

3단계: 컴퓨터 재부팅 후 재설치
시스템이 환경 변수 변경 사항을 완전히 적용하도록 컴퓨터를 한 번 재부팅해 줍니다.

그 후 다운로드해 둔 설치 파일을 실행하여 설치를 진행하시되, "추가 작업 선택" 단계에서 PATH에 추가 옵션이 잘 체크되어 있는지 꼭 다시 확인하고 설치를 완료해 주세요.

💡 만약 이렇게까지 했는데도 안 된다면?
완전 재설치 후에도 안 된다면, 현재 사용 중이신 윈도우 계정의 권한 문제이거나 PowerShell 자체의 보안 정책(Execution Policy) 이슈일 수 있습니다. 이럴 때는 아쉽더라도 복잡하게 윈도우 시스템을 만지는 것보다 방법 2(PowerShell 프로필 등록)를 사용하시는 것이 정신 건강과 개발 생산성 측면에서 가장 빠르고 스트레스 없는 해결책이 될 수 있습니다.
